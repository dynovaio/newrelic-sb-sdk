REGLAS DE FORMATO:

2. Agrupa los cambios estrictamente por tipo de objeto GraphQL, usando exactamente estas categorías si aplican:
   - "Add new scalar:"
   - "Add new enums:"
   - "Add new input objects:"
   - "Add new objects:"
   - "Update existing types:"
   - "Add mutations to `RootMutationType`:"
3. Lista los elementos dentro de cada categoría alfabéticamente.
4. Usa el estilo de lista con viñetas (*) para las categorías principales.
5. Para listas largas de elementos (como en Enums), usa comas para separarlos en una sola entrada si es posible, o sub-listas si es necesario para claridad.
6. NO inventes información. Basa todo estrictamente en el código añadido/modificado en el diff.
7. Genera SOLAMENTE el markdown, sin texto conversacional antes o después.
